## Next Feature Idea
**Title:** Create idea or ticket from run history
**Description:** Let users turn any completed run in the Worker tab’s History into a planning artifact. From the History section, add actions “Create idea from this run” and “Add as ticket” so the run’s label and output can be saved as an idea (in the Ideas pool) or as a Planner ticket, optionally with an AI-generated title and description. This closes the loop from execution back to planning without manual copy-paste.
**Rationale:** The app already has rich run history (output, label, duration) and full support for ideas and tickets in the DB and Planner. There is no path today from “this run was useful” to “record it as an idea or ticket.” Adding this flow fits the existing architecture (run-store, createIdea/createTicket, project context), delivers clear user value (reuse runs as planning artifacts), and can be implemented in a few milestones (UI actions, optional summarize API, linking to current project).

## Project Summary
KWCode is a Tauri 2 + Next.js 16 desktop app for managing repositories and running Cursor/Claude/Gemini CLI agents from one dashboard. It offers a Projects list (add repo, Initialize, Starter), a Worker tab (Ask, Plan, Fast dev, Debug with terminal output and history), a Planner (Kanban, milestones, tickets, sync with `.cursor/7. planner/tickets.md`), and per-project or global Ideas, Prompts, Design, and Architecture. Data lives in SQLite and file-based storage (`data/`, `.cursor/`). The “analyze project” flow writes a structured idea document to `data/prompts/idea-analysis-output.md`, which the Run tab parses to drive idea-driven flows; ideas can be improved via API and turned into full projects (prompts, tickets, design, architecture) with “Generate project from idea.”

## Implementation Considerations
- **Where it lives:** Run tab History section in `ProjectRunTab.tsx` (e.g. per-entry dropdown or buttons: “Create idea,” “Add as ticket”). Reuse existing `createIdea` and `createTicket` from `src/lib/data/ideas.ts` and `src/lib/data/tickets.ts`; link new ticket/idea to current project via project’s `ticketIds` / idea association if the data model supports it.
- **Optional AI summarization:** Add an API route (e.g. `POST /api/data/run-to-idea` or under projects) that takes run output and returns `{ title, description }` via the existing agent runner (`runAgentPrompt`), so long outputs become concise idea/ticket text. First milestone can ship without AI (use run label as title, first N chars of output as description).
- **Dependencies:** run-store’s `terminalOutputHistory`, project context from props/params, no new backend storage beyond existing DB.
- **Risks:** Very long output may need truncation or summarization for ticket/idea description; linking “run → idea/ticket” for traceability could be a follow-up (e.g. optional `sourceRunId` on idea/ticket if schema is extended).
